Introduction

[back to top]

The purpose of this feature is to allow users to adorn their diagram with images such as GIF / JPG / SVG. The user may wish to demonstrate some information that is not part of the semantic model, either to enhance the cosmetic appeal of the diagram or to help further the point of the diagram to trying to convey. The mechanism to do this would be through copy / paste from an external application or by dragging a URI that points to an image file onto the diagram.

Requirements

[back to top]

Import image/graphics into the model project
Drag & Drop of image/graphics onto the drawing surface
Copy & past images onto the drawing surface
Minor manipulation (resizing and/or cropping) of the image/graphic on the drawing surface
Images added as an icon or as a background to the drawing surface
UML elements can appear on top or underneath of the image
Images should be inserted by reference. If an image is used in multiple diagrams & then the image file content is changed, the changes automatically appear in all diagrams where it is used

Constraints and Limitations

[back to top]

Copy / paste support won't work on Linux due to an existing bugzilla. See <https://bugs.eclipse.org/bugs/show_bug.cgi?id=70437>
Copy / paste of WMF/EMF is not supported because there is no vector based transfer flavor supported by JDK 1.4.2. Only image or bitmap based formats can be pasted from outside applications.

Drag and drop an image file onto the diagram surface

[back to use cases]

The user wishes to add an image / picture to the drawing surface.

Preconditions:

  • An image file exists on the file system and is in one of the following graphic formats: BMP, JPG, GIF, SVG, WMF
  • The OS supports drag/drop of files.

Flow of events:

  1. The user exports from an outside application to an existing file format.
  2. The user selects the file in the OS file navigator and drags it onto the RMP drawing surface.
  3. The cursor changes to an arrow + cursor indicating the drop is permitted.

Postconditions:

  • The file is displayed as an image on the drawing surface and appears selected.

Note: By default this file will loaded into a buffer and stored explicitely inside the notation file. However, clients may wish to override this behavior to suport a by reference storage mechanism. For instance, in the UML product offering the underlying will be reinterpreted as a UML comment which references the file. However, in this case if the model is transported, then the referenced file would have to be as well in order for the image to display.

Drag and drop a non-image image onto the diagram surface

  • [back to use cases]

    The user wishes to add an image / picture to the drawing surface.

    Preconditions:

    • A file exists on the file system that is in NOT of the following graphic formats: BMP, JPG, GIF, SVG, WMF
    • The OS supports drag/drop of files.

    Flow of events:

    1. The user exports from an outside application to an existing file format.
    2. The user selects the file in the OS file navigator and drags it onto the RMP drawing surface.
    3. The cursor changes to an arrow + cursor indicating the drop is permitted.

    Postconditions:

    • The file is displayed as a note on the drawing surface and appears selected.
    • The note contains the path of the file.

Note: In the UML product offering the note will be reinterpreted as a UML comment. This allows for the double click gesture where the associated external editor is invoked for the particular file extension.

Copy and paste an image from an outside application

  • [back to use cases]

    The user wishes to add an image / picture to the drawing surface.

    Preconditions:

    • An application exists in the OS that can copy selections to an image on the clipboard (i.e. Paint)
    • The OS supports image copy.

    Flow of events:

    1. The user copies a selection from an outside application.
    2. The user navigates to the RMP diagram surface, right-clicks on the background and choses, Edit / Paste...

    Postconditions:

    • An image is displayed on the drawing surface and appears selected..

Note: Copy / paste from outside applications is not a by reference operation. Multiple pastes will increase the file size by the size of the image buffer on the clipboard..

Resizing an image on the diagram surface

  • [back to use cases]

    The user wishes to resize on existing image on the drawing surface.

    Preconditions:

    • The image shape is selected on the drawing surface..

    Flow of events:

    1. The user grabs the selection handles on the image shape and drags to the desired size..

    Postconditions:

    • The image is rerendered at the new size specified by the handle feedback..


Future considerations:
- Control over brightness and contrast.
- Cropping functionality - ability to crop - top, left, right or bottom.
- Tiled image support. Specifying a tile size and offset and have the image repeat itself within a shape. This is useful for backgrounds.
